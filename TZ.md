Разработчик: Потапов Максим Алексеевич РИ-241001

# Цель проекта и концепция
Тема: Аналитическая веб-система для музыкальных продюсеров, диджеев и исследователей поп-культуры с целью визуализации и отслеживания исторических цепочек музыкального семплирования (заимствования звука).

Проблематика: История современной музыки тесно связана с семплированием, однако отследить первоисточник конкретного звука, бита или вокала бывает сложно. Проект решает эту задачу, сощдавая граф связей между треками.

# Роли пользователей
Гость (Основная аудитория): Имеет доступ к публичному интерфейсу аналитики. Может использовать динамические фильтры каталога для перестроения графа под конкретного исполнителя, изучать связи заимствований и переходить по внешним ссылкам для прослушивания оригинала и нового трека.
Администратор (Контент-менеджер): Имеет защищенный доступ к панели управления Django Admin. Управляет базой данных: добавляет новых артистов, треки и фиксирует связи между ними, указывая конкретный тип заимствованного семпла (вокал, мелодия, ударные).

# Модели данных (Сущности)
В проекте реализована реляционная база данных, состоящая из 3-х связанных кастомных моделей:

Модель 1: `Artist` (Исполнитель)
  Поля: `name` (Имя/Название), `era` (Период активности).
  Назначение: Хранение уникальных артистов для каталога и работы динамического фильтра.
Модель 2: `Track` (Музыкальный трек)
  Поля: `title` (Название), `artist` (Связь ForeignKey с Artist), `listen_link` (Опциональная URL-ссылка на прослушивание).
  Назначение: Хранение аудиозаписей, привязанных к конкретным авторам (Связь "Один ко многим").
Модель 3: `SampleRelation` (Связь заимствования / Семпл)
  Поля: `source_track` (ForeignKey к Track — откуда взяли звук), `target_track` (ForeignKey к Track — куда вставили звук), `sample_type` (Категория семпла).
  Назначение: Узловая таблица, описывающая направленную связь между двумя разными треками. Позволяет строить аналитические цепочки любой длины.

# Ключевой функционал (User Stories)
Динамическая фильтрация каталога: Пользователь кликает на имя артиста в интерфейсе. Идет отправка GET-запроса `?artist_id=`, контроллер фильтрует все связи в базе данных (используя объекты `Q` в Django ORM) и оставляет только релевантные треки для конкретного исполнителя.
Аналитическая визуализация (Генерация графа): На основе отфильтрованных данных система автоматически вычисляет координаты объектов и строит интерактивный сетевой граф связей. Пользователь может наводить курсор на узлы графа для получения дополнительной информации.
Внешняя навигация: В списке найденных связей пользователь видит интерактивные значки ссылок. При нажатии открывается новая вкладка браузера с источником (YouTube) для прямого сравнения звучания.

# Внутренняя аналитика и технологии
Проект удовлетворяет критерию "Глубокая внутренняя аналитика", так как реализует сложный модуль обработки данных.
Pandas: Используется для трансформации сырых данных из Django ORM в структурированный DataFrame, что позволяет эффективно подготавливать данные для отрисовки.
NumPy: Применяется для сложных тригонометрических расчетов (вычисление синусов и косинусов `np.linspace`) с целью равномерного математического распределения координат узлов графа на плоскости.
Plotly Graph Objects: Продвинутая библиотека визуализации, которая на основе датафрейма и координат генерирует интерактивный HTML-код графа с поддержкой hover-эффектов и автомасштабирования.
